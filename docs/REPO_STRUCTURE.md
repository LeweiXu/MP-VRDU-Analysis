# Repo structure

The science spine sits flat at the root; operational tooling groups under `ops/`.
A file's home is decided by what it is, not who imports it.

```
mpvrdu/
├── config.py                 run knobs: paths, resolution presets, sampling defaults
├── schema.py                 frozen data contracts + telemetry fields
│
├── data/                     dataset layer
│   ├── loader.py             rows -> Question; answerable/unanswerable split
│   ├── binning.py            bin_label lookup from manual annotations
│   ├── annotations.py        read/validate the human doc-label table
│   └── render.py             PDF -> per-page PNG + embedded-text spans
│
├── tools/                    per-page channel builders
│   ├── text.py               cheap embedded text (T)
│   ├── parser.py             PDF-parser markdown for TL/TLV
│   └── visual.py             page image + vision-token estimation
│
├── retrievers/               page retrievers (cost rungs)
│   ├── text.py               BM25 / BGE-M3 / Qwen3-Embedding
│   ├── vision.py             ColModernVBERT / ColQwen2.5 / ColQwen3
│   └── joint.py              union of a text + a vision page set
│
├── models/                   reasoner backends + classifier, behind registries
│   ├── __init__.py           get_reasoner registry
│   ├── qwen3vl.py            Qwen3-VL backend
│   ├── internvl.py           InternVL backend
│   ├── classifier.py         doc-type/bin classifier
│   └── payload.py            backend-neutral ModelInput
│
├── pipeline/                 the five frozen stages of one cell
│   ├── conditioner.py        page selection (oracle / retrieved / similarity)
│   ├── representation.py     T/TL/TLV/V composer (cost-ordered; parser text; no bbox)
│   ├── reasoner.py           Reasoner ABC
│   ├── judge.py              scoring interface + API judges
│   └── orchestrator.py       composes stages; owns caches + telemetry
│
├── scoring/                  cached cells -> numbers
│   ├── accuracy.py           doc-level accuracy + bootstrap CIs
│   ├── cost.py               token / latency / VRAM aggregation
│   ├── frontier.py           sufficiency-frontier rule
│   ├── retrieval.py          page precision / recall / F1
│   ├── abstention.py         abstention detection
│   └── agreement.py          judge-human kappa
│
├── experiments/              what runs / how / on what
│   ├── tasks/                G[num]_[name] generation tasks + base
│   ├── engine/               driver, side artifacts, artifacts, paths
│   ├── corpus/               resolve, smoke, yaml_spec
│   └── registry.py           task-name -> task collection
│
├── reporting/                judged rows -> tables
│   ├── build.py              task->table routing + build-time routing assembly
│   └── tables/               content-named builders (headline, parser, ...)
│
├── ops/                      entry points + operational tooling
│   ├── generate.py judge.py build.py   the three role entry points
│   ├── kaya/                 SLURM runner + config + guides
│   ├── specs/                YAML specs (template + saved run configs)
│   └── scripts/              standalone utilities
│
├── docs/                     authored prose + generated/ script outputs
├── tests/                    pytest suite + fixtures/
└── old/            [reference]  v3 snapshot; deleted when v4 is green
```
